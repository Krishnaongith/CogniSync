SLIDE 4 — INTRODUCING COGNISYNC
====================================================
Headline: CogniSync — your academic co-pilot.

What it does (one sentence each):
- Upload any academic document (PDF, DOCX, PPTX, XLSX, TXT) or paste text directly.
- AI extracts the key points you actually need to know.

- Lets you choose how you read: one concept at a time (Focus View) or numbered steps (Step-by-Step View).

SLIDE 6 — AI INTEGRATION (DEEP DIVE)
====================================================
Headline: AI isn't a feature. It's the foundation.

How AI powers CogniSync:

1. Model: Amazon Nova Pro (via AWS Bedrock) — chosen for its strong instruction-following, JSON output reliability, and cost efficiency at scale.

2. Custom system prompt engineering:
   - Single prompt returns structured JSON with keyPoints, tasks (with urgency/importance tags), and rewrittenText — all in one call.
   - Profile-aware: the system prompt dynamically changes tone and style based on the user's selected cognitive profile (ADHD = short punchy sentences, Anxiety = no urgency language, Dyslexia = simple words and calm tone).
   - Strict output rules: "Return ONLY the JSON object, no markdown, no explanation."

3. Two AI endpoints:
   - /process — full document analysis: key points + task extraction + plain language rewrite in one shot.
   - /rewrite — real-time complexity dial: rewrites content at any grade level (1–16) on demand, profile-aware tone.

4. Client-side Flesch-Kincaid scoring — runs entirely in the browser, no API call needed. Computes before/after complexity scores and percentage reduction.

5. Eisenhower Matrix auto-classification — AI tags each task with urgency and importance, which the frontend maps to quadrants automatically.

6. Mock/live architecture — a single config flag switches between a realistic mock layer (for offline/demo use) and the live Bedrock API. No code changes needed in consuming components.

Visual suggestion: Architecture diagram showing: User → Frontend → Node.js Server → AWS Bedrock (Nova Pro) → back to Frontend. Show the two endpoints and the client-side scorer as a separate branch.

====================================================
SLIDE 7 — TECHNICAL EXECUTION
====================================================
Headline: Built to work. Built to scale.

Stack:
- Frontend: React + TypeScript + Vite
- Backend: Node.js + Express
- AI: AWS Bedrock (Amazon Nova Pro v1)
- File parsing: PDF, DOCX, PPTX, XLSX, TXT — up to 100 MB
- Scoring: Flesch-Kincaid algorithm implemented client-side in TypeScript

Custom logic highlights:
- Layered processor architecture: UI → Ingestion → Processor (API gateway) → Mock or Live API → Scorer → UI. Consuming components never know which mode is active.
- Property-based testing with fast-check: 16 correctness properties verified across 100+ iterations each (file size boundaries, key point count invariants, task structure, focus view navigation bounds, mock delay timing, etc.)
- Adaptation profile system: 4 profiles (Default, ADHD, Dyslexia, Anxiety) each with configurable chunk size, whitespace, tone — applied both to AI prompt and to CSS rendering.
- Complexity dial: real-time rewrite at any of 16 grade levels (Kindergarten → Graduate), debounced, with profile-aware tone.
- Eisenhower Matrix: tasks auto-classified into 4 quadrants by urgency × importance, with per-quadrant task counts and completion animations.
- Zero persistent storage: all state is session-scoped, no user data retained.

Visual suggestion: Code snippet showing the system prompt with profile injection. Or the architecture diagram from Slide 6.

SLIDE 8 — DESIGN & UI/UX
====================================================
Headline: Designed for the brain, not just the eye.

Design principles:
- Calm, modern, trustworthy — Linear meets Notion meets a wellness app.
- Every design decision serves cognitive accessibility, not just aesthetics.

Key UX decisions:
- Card-based layout: each output section is a distinct, scannable card with a colored left-border accent. No wall of text.
- Adaptation profiles: user sets their needs once, the entire output adapts — font size, line height, letter spacing, tone, chunk size.
- Focus View: one concept at a time, with direction-aware animations and a progress indicator ("3 of 8"). Designed for students who get overwhelmed by long lists.
- Complexity Dial: a gradient slider (green → red) that rewrites content at any grade level in real time. Satisfying to use, immediately understandable.
- Priority Matrix: color-coded Eisenhower quadrants (red = Do Now, green = Schedule, amber = Delegate, slate = Eliminate) with axis labels and task count badges.
- Skeleton loaders: animated shimmer placeholders during processing — no jarring layout shifts.
- Accessibility: skip-to-main link, aria-live regions for processing status and character count, focus-visible rings on all interactive elements, prefers-reduced-motion respected throughout, no outline:none without replacement.

Visual suggestion: Side-by-side of the app in Default vs. ADHD vs. Dyslexia profiles showing the visual difference. Or a close-up of the Priority Matrix and Complexity Dial.

SLIDE 9 — SOCIAL GOOD & ACCESSIBILITY
====================================================
Headline: Equity in education starts with accessible content.

The social case:
- Hidden cognitive differences are underdiagnosed, especially in underserved communities where formal accommodations are rare or unavailable.
- CogniSync requires no diagnosis, no IEP, no formal accommodation request. Any student can use it, right now.
- It works on any device with a browser — no app install, no account required.
- It is free to use at the point of need.

Accessibility built in, not bolted on:
- WCAG AA contrast ratios throughout.
- Full keyboard navigation.
- Screen reader support with semantic HTML and ARIA landmarks.
- Respects prefers-reduced-motion for users with vestibular disorders.
- No user-scalable=no — users can zoom freely.
- Profiles designed with input from documented needs of ADHD, dyslexia, and anxiety communities.

Path to adoption:
- Phase 1: Direct student use (current MVP).
- Phase 2: Integration with LMS platforms (Canvas, Blackboard) via browser extension.
- Phase 3: Institutional licensing for universities and community colleges.
- Phase 4: API for ed-tech platforms to embed CogniSync's simplification pipeline.

SLIDE 10 — IMPACT & SCALE

Immediate beneficiaries:

- First-generation college students navigating academic language for the first time.

SLIDE 11 — INNOVATION
====================================================
Headline: Not just simpler text. A new way to experience academic content.

What makes CogniSync original:

1. Profile-aware AI prompting: the same document produces genuinely different outputs depending on the user's cognitive profile — not just a font change, but a different tone, sentence structure, and pacing from the model itself.

2. Real-time complexity dial: drag a slider, get a live rewrite at any grade level. This is not a static simplification — it's a continuous spectrum from Kindergarten to Graduate, on demand.

3. Eisenhower Matrix from AI: tasks aren't just listed — they're classified by urgency and importance by the model, then rendered as a visual productivity matrix. No other academic tool does this.

4. Before/after cognitive load scoring: Flesch-Kincaid computed client-side on both the original and simplified text, with a live percentage reduction. Makes the value of simplification tangible and measurable.

5. Dual reading modes designed for cognitive differences: Focus View (one concept at a time) and Step-by-Step View are not just UI preferences — they are evidence-based accommodations for attention and processing differences.

6. Zero-barrier access: no account, no diagnosis, no accommodation request. Open the app, paste your syllabus, get help.
